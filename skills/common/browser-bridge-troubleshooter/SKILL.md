# 通用浏览器桥接故障诊断

当项目通过本地服务、浏览器插件、WebBridge、QWebBridge 类工具、Playwright 连接器或自定义桥接方案操控真实浏览器页面时，使用这个 skill。

## 核心原则

按层定位问题。不要把所有网页自动化失败都直接归咎为应用代码错误。

## 分层排查顺序

1. 应用集成层：
   - 应用是否调用了正确的桥接提供方。
   - 地址、端口、超时时间、模式是否正确。
2. 本地服务层：
   - 桥接服务是否启动。
   - 健康检查接口是否正常。
   - 服务版本是否和应用兼容。
3. 浏览器插件层：
   - 插件是否安装。
   - 插件是否启用。
   - 插件是否显示已连接或已就绪。
   - manifest 是否能正常加载。
4. 目标网页层：
   - 是否已登录。
   - 是否还在加载、验证或风控。
   - 当前标签页是否正确。
5. 输入层：
   - 是否能找到输入框。
   - 输入后文本是否真实出现在页面里。
6. 发送层：
   - Enter 是否能发送。
   - 配置的发送按钮 selector 是否有效。
   - 按钮是否可见、可点、未禁用。
7. 等待层：
   - 是否等到回答生成完成。
   - 是否存在停止生成或加载状态。
   - 回答文本是否已经稳定。
8. 抓取层：
   - 是否能抓到 DOM 文本。
   - 是否能抓到来源链接。
   - 截图是否覆盖有效证据区域。

## 常见故障模式

- HTTP 502/503：本地服务存在，但浏览器插件或页面命令执行失败。
- 健康检查失败：服务没启动、端口不对、地址不对或被拦截。
- 守护进程假死：PID 文件还在，但实际进程不存在。
- 日志轮转失败：启动时报 `rotate log` 或 `Access is denied`，通常需要清理旧日志或 stale PID。
- 输入成功但发送失败：selector 错误、按钮禁用、Enter 行为变化。
- 发送成功但无回答：未登录、验证码、风控、敏感拦截或等待时间太短。
- 回答只抓到一部分：容器选错、虚拟滚动、滚动位置错误或视口截图限制。
- 截图没截到证据：需要回答容器截图、整页截图或提及位置截图。

## 发送按钮定位策略

按钮点不到时按顺序尝试：

1. 使用已配置 selector。
2. 尝试语义 selector：role、aria-label、title、文本。
3. 扫描输入框附近的可见按钮。
4. 返回候选 selector、按钮标签和置信度。
5. 如果不能确认发送成功，应停止当前批次，不继续下一题。

## 等待回答策略

优先用完成状态判断，不要只靠固定秒数：

- 回答容器出现。
- 文本长度多次轮询不再变化。
- 加载或停止按钮消失。
- 发送按钮恢复可用。
- 用户设置的等待时间只作为上限。

## 抓取策略

优先保存证据，而不是只追求截图好看：

1. 完整回答文本。
2. 回答区域内的信息来源链接。
3. 回答容器截图。
4. 如果完整容器太长，截图关键证据区域。
5. 如果关注品牌或实体提及，定位提及文本并截图上下文。

长回答场景不要只依赖当前视口截图。优先保存完整文本；截图则尽量截回答容器，截不全时至少截品牌首次提及区域和回答结尾区域。

## 服务恢复策略

如果本地桥服务无法连接：

1. 先查状态命令或健康检查接口。
2. 确认 PID 是否对应真实进程。
3. 如遇日志文件锁，备份或移动旧日志后再启动。
4. 服务启动后如果插件未连接，等待数秒并打开真实网页触发扩展连接。
5. 不要在排障中途随意升级桥服务版本，除非错误明确要求升级。

## 错误提示标准

错误信息要说明失败层级：

- 本地服务不可达。
- 浏览器插件未就绪。
- 页面未登录。
- 输入框 selector 失败。
- 发送按钮 selector 失败。
- 等待回答超时。
- 回答或截图抓取失败。

不要只显示“自动化失败”或原始 `[object Object]`。

## 验证清单

- 健康检查通过。
- 插件显示就绪。
- 目标网页能打开。
- 问题能输入。
- 发送动作能确认成功。
- 能识别回答生成完成。
- 能保存完整回答或关键证据。
- 失败时能安全停止当前批次。
