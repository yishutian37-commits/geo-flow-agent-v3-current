# PRD Gap Implementation

用于根据 `通用GEO流程Agent_PRD_V2.3.docx` 或其他 PRD，对照当前项目实现状态，找出未完成能力并按优先级逐步补齐。

## 工作方式

不要一次性泛泛列清单。按“读取 PRD -> 对照代码 -> 排优先级 -> 单轮施工 -> 测试 -> 汇报”的节奏推进。

## 固定流程

1. 读取 PRD：
   - 提取模块
   - 提取页面
   - 提取字段
   - 提取业务流程
   - 提取异常状态
2. 对照当前实现：
   - 后端模型
   - API endpoint
   - service 逻辑
   - 前端页面
   - 前端 API 封装
   - 桌面打包入口
3. 标注状态：
   - 已实现
   - 部分实现
   - 未实现
   - 实现偏离 PRD
   - 已实现但体验不可用
4. 排优先级：
   - P0：影响主流程闭环或数据可信度。
   - P1：影响业务效率和报告价值。
   - P2：增强体验或自动化程度。
5. 每轮只做一个清晰功能包。
6. 每轮完成后必须测试。

## 优先级判断标准

P0：

- 项目/品牌/事实/问题/检测/内容/报告主链路断开。
- 数据保存后不能复用。
- 检测结果不可信或不可追溯。
- 发布或报告无法形成闭环。

P1：

- 多平台、多批次、多报告聚合能力不足。
- 记忆库、反馈、来源资产分析不完整。
- 页面能用但业务人员看不懂。

P2：

- 自动发布。
- 定时检测。
- 竞品深度分析。
- 高级图表和评分。

## 测试要求

每轮至少执行相关测试：

- 后端变更：`.\.venv\Scripts\python.exe -m pytest tests -q`
- 前端变更：`npm.cmd --prefix frontend run build`
- 桌面变更或用户要求：`npm.cmd run dist:win`

## 汇报格式

每轮结束后输出：

- 本轮补齐的 PRD 条目
- 修改的模块
- 验证结果
- 新增或更新的测试
- 剩余未实现项
- 下一步建议
