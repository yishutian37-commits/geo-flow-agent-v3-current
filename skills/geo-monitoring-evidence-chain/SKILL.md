# GEO Monitoring Evidence Chain

用于处理 GEO 监测中心、AI 搜索详情、信息来源抓取、检测明细表、报告聚合、复测闭环相关任务。

## 核心目标

把系统从“单次自动检测工具”收束为“问题库驱动的 GEO 监测、来源分析、内容优化闭环系统”。

标准链路：

`品牌事实 -> 问题库 -> 多平台检测 -> 原始回答留档 -> 推荐/提及/情绪/来源分析 -> 趋势仪表盘 -> 内容优化任务 -> 发布记录 -> 复测`

## 数据证据标准

每条检测样本至少应能追溯：

- 项目 ID
- 问题 ID 和问题文本
- 平台 ID 和平台名称
- 检测批次 ID
- 原始回答全文
- 是否提及品牌
- 是否推荐品牌
- 情绪判断
- 判断依据
- 信息来源链接列表
- 截图 URL
- 检测时间

其中“原始回答全文”和“品牌命中证据”是核心证据，不得只保存最后一个视口、模型摘要或 JSON 分析结果。

## 监测明细标准

推荐主表按“问题 x 平台”展示，而不是只按批次展示。

表格字段建议：

- 问题
- 平台
- 提及品牌
- 推荐状态
- 情绪
- 信息来源数
- 检测时间
- 操作：AI 搜索详情、删除

## AI 搜索详情标准

详情页必须展示：

- 问题
- AI 平台
- 原始回答全文
- 品牌提及证据
- 是否推荐和判断依据
- 情绪判断
- 信息来源列表
- 回答截图或品牌提及位置截图
- 检测时间

如果完整截图困难，优先定位品牌提及片段并截图该区域，同时保留尽可能完整的回答文本。

用户看到的回答正文必须是自然文本；如果视觉模型或分析模型返回 JSON，只能作为内部结构化结果保存，不要直接展示给业务用户。

长回答不要只保存最后一个视口。证据保存优先级是：

1. 完整回答文本。
2. 品牌首次提及位置和上下文。
3. 品牌提及区域截图。
4. 回答结尾截图。
5. 当前视口截图。

如果系统只能截到后半段，但品牌在前半段出现，应先用文本定位品牌位置，再滚动或定位到该区域截图。

## 文本模式与视觉模式

默认使用文本 DOM 抓取模式。只有用户明确开启视觉识别模式时，才使用多模态模型分析截图。

文本模式要求：

- 优先抓回答容器文本，不要抓整页 UI 噪声。
- 过滤“你需要什么帮助”“参考 0”“深度分析需求并解答”等平台提示语。
- 如果完整回答很长，保留完整文本；页面只展示可读摘要和详情入口。
- 品牌匹配要支持公司全称、简称、别名、去括号名称和事实库品牌名。

视觉模式要求：

- 只有模型配置明确支持图片理解时才启用。
- 截图应覆盖完整回答容器；过长时至少覆盖品牌首次出现位置、上下文和结尾。
- 模型输出必须能解析为结构化结果；解析失败时保留原始输出并给出可读错误。
- 视觉模式失败时不要伪造结论，应提示用户切回文本模式或更换多模态模型。

## WebBridge 抓取排障

遇到“网页已回答但系统没抓到”时，按顺序检查：

1. 是否真的发送成功，输入框是否还保留原问题。
2. 等待策略是否只靠固定秒数，应该增加文本稳定判断。
3. 回答容器 selector 是否选到了输入区、侧边栏或历史记录。
4. 平台是否使用虚拟滚动，导致 DOM 只有当前片段。
5. 品牌是否只以简称出现，需要扩展品牌别名。
6. 截图是否只截了当前视口，需要回答容器或命中片段截图。
7. 信息来源是否在回答容器内的 `<a href>`，不是页面全局链接。

## 信息来源抓取标准

WebBridge 抓取回答后，应同时抓回答容器中的链接：

```json
{
  "title": "来源标题",
  "url": "https://example.com/page",
  "source_type": "ai_citation",
  "is_own_asset": false,
  "is_competitor": false
}
```

来源分析要区分：

- 自有资产：官网、公众号、百家号、小红书、新闻稿。
- 权威资产：政府、协会、教育平台、媒体。
- 竞品资产。
- 低质量聚合页。
- 无来源。

## 修复优先级

P0：

- 原始回答完整保存。
- 品牌提及证据可定位。
- 信息来源链接可保存。
- 明细表按问题和平台展示。
- 删除检测明细时清理关联数据和截图文件。
- 视觉识别模式不能只分析最后一屏，必须尽量围绕完整回答或品牌提及位置取证。
- 检测明细必须支持删除，并清理关联来源、截图、分析结果。

P1：

- 来源资产分析。
- 多批次报告聚合。
- 平台推荐率对比。
- 趋势图。

P2：

- 竞品来源监测。
- 自动生成内容优化任务。
- 发布后复测闭环。
- 定时监测。

## 验证清单

- 单平台单问题检测能生成一条明细。
- 多平台批量检测能区分每个平台。
- 详情页能看到原始回答和截图。
- 品牌出现在回答开头、中间、结尾时都能识别提及。
- 截图至少覆盖品牌提及上下文或完整回答容器。
- 有链接的平台能抓到来源。
- 删除样本后页面刷新，数据库和截图文件同步清理。
- 报告能聚合多组检测记录。
