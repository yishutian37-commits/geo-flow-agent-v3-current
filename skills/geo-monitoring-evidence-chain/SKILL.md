# GEO Monitoring Evidence Chain

用于处理 GEO 监测中心、AI 搜索详情、信息来源抓取、检测明细表、报告聚合、复测闭环相关任务。

## 核心目标

把系统从“单次自动检测工具”收束为“问题库驱动的 GEO 监测、来源分析、内容优化闭环系统”。

标准链路：

`品牌事实 -> 问题库 -> 多平台检测 -> 原始回答留档 -> 推荐/提及/情绪/来源分析 -> 趋势仪表盘 -> 内容优化任务 -> 发布记录 -> 复测`

## 数据证据标准

每条检测样本至少应能追溯：

- 项目 ID
- 问题 ID 和问题文本
- 平台 ID 和平台名称
- 检测批次 ID
- 原始回答全文
- 是否提及品牌
- 是否推荐品牌
- 情绪判断
- 判断依据
- 信息来源链接列表
- 截图 URL
- 检测时间

## 监测明细标准

推荐主表按“问题 x 平台”展示，而不是只按批次展示。

表格字段建议：

- 问题
- 平台
- 提及品牌
- 推荐状态
- 情绪
- 信息来源数
- 检测时间
- 操作：AI 搜索详情、删除

## AI 搜索详情标准

详情页必须展示：

- 问题
- AI 平台
- 原始回答全文
- 品牌提及证据
- 是否推荐和判断依据
- 情绪判断
- 信息来源列表
- 回答截图或品牌提及位置截图
- 检测时间

如果完整截图困难，优先定位品牌提及片段并截图该区域，同时保留尽可能完整的回答文本。

## 信息来源抓取标准

WebBridge 抓取回答后，应同时抓回答容器中的链接：

```json
{
  "title": "来源标题",
  "url": "https://example.com/page",
  "source_type": "ai_citation",
  "is_own_asset": false,
  "is_competitor": false
}
```

来源分析要区分：

- 自有资产：官网、公众号、百家号、小红书、新闻稿。
- 权威资产：政府、协会、教育平台、媒体。
- 竞品资产。
- 低质量聚合页。
- 无来源。

## 修复优先级

P0：

- 原始回答完整保存。
- 品牌提及证据可定位。
- 信息来源链接可保存。
- 明细表按问题和平台展示。
- 删除检测明细时清理关联数据和截图文件。

P1：

- 来源资产分析。
- 多批次报告聚合。
- 平台推荐率对比。
- 趋势图。

P2：

- 竞品来源监测。
- 自动生成内容优化任务。
- 发布后复测闭环。
- 定时监测。

## 验证清单

- 单平台单问题检测能生成一条明细。
- 多平台批量检测能区分每个平台。
- 详情页能看到原始回答和截图。
- 有链接的平台能抓到来源。
- 删除样本后页面刷新，数据库和截图文件同步清理。
- 报告能聚合多组检测记录。
