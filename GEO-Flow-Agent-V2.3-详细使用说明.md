# GEO Flow Agent V2.3 详细使用说明

本文档面向实际使用者，说明 GEO Flow Agent V2.3 的安装、初始化、资料录入、问题库生成、文章生成、监测分析、报告中心、浏览器插件安装与常见故障处理。

当前项目目录：

`D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2`

当前桌面应用包目录：

`D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\release`

## 1. 这个 Agent 是做什么的

GEO Flow Agent 是一个本地运行的 GEO/AIO 生成式引擎优化工作台，核心目标是帮助企业围绕 AI 搜索和 AI 问答产品完成：

1. 项目管理：建立企业/品牌项目。
2. 品牌事实库：把企业资料沉淀成可引用、可审核的事实。
3. 资料缺口诊断：发现资质、地址、产品、案例、价格等资料缺口。
4. 问题库生成：围绕用户会问 AI 的问题建立问题矩阵。
5. 内容任务与文章生成：基于问题、事实、记忆规则生成文章草稿。
6. 发布与记录：管理稿件、发布渠道和发布记录。
7. 监测分析：通过浏览器插件自动打开 AI 产品网页，输入问题，抓取回答并统计品牌提及率、推荐率等指标。
8. 报告中心：生成客户报告或内部报告，支持多组检测记录聚合。

## 4. 配置大模型 API

文章生成、事实提取、问题库生成、反馈记忆分析，都需要配置大模型 API。

操作路径：

`左侧菜单 -> AI模型`

建议配置方式：

1. 进入 AI 模型页面。
2. 新增模型或编辑已有模型。
3. 填写：
   - 模型供应商
   - 模型名称
   - API Key
   - Base URL
4. 点击测试。
5. 测试通过后，设置为默认模型。

注意：

- 如果没有默认模型，文章生成、问题库生成、事实提取会失败。
- 如果模型响应很慢，文章生成可能需要 1-3 分钟。
- 当前文章生成相关接口已经放宽到 5 分钟超时。
- 若仍然超时，可以换响应更快的模型，或降低文章长度要求。

操作路径：

项目创建后，可以在项目列表中查看、编辑或删除项目。

## 6. 添加企业资料和品牌事实

品牌事实是整套系统的基础。文章生成、监测分析、报告结论都应该尽量基于已确认事实。

操作路径：

`左侧菜单 -> 品牌事实库`

推荐流程：

1. 先选择项目。
2. 找到“批量资料/AI提取”相关入口。
3. 把企业资料一次性粘贴进去，例如：
   - 企业介绍
   - 资质证书
   - 证书编号
   - 地址
   - 联系方式
   - 课程/产品信息
   - 价格
   - 案例
   - 荣誉
   - 官网、公众号、媒体报道
4. 点击 AI 提取。
5. 系统会把大段资料拆成事实候选。
6. 人工检查每条事实。
7. 对确认无误、可以公开使用的事实点击确认。

重要概念：

- `draft`：待确认事实，不能直接作为正式公开依据。
- `confirmed`：已确认事实，可以被文章和报告引用。
- `public`：可公开使用。
- `restricted`：限制使用，不应该进入公开文章。

如果文章生成提示“当前项目没有可公开使用的已确认品牌事实”，说明这个项目还没有足够的 `confirmed + public` 事实，需要先确认资质、产品、地址、案例等基础事实。

## 7. 资料缺口诊断

操作路径：

`项目详情 -> 资料缺口诊断`

用途：

帮助判断当前项目缺哪些资料，例如：

- 企业主体是否明确
- 品牌名是否明确
- 资质证书是否缺编号
- 地址是否缺详细门牌
- 产品/服务是否缺价格或交付说明
- 是否缺案例、荣誉、联系方式

建议使用方式：

1. 先录入企业资料。
2. 提取并确认一批品牌事实。
3. 再运行资料缺口诊断。
4. 根据诊断结果补资料。

## 8. 生成问题库

问题库用于模拟用户会向 AI 产品提出的问题，是 GEO 优化的起点。

操作路径：

`项目详情 -> 问题库 -> 生成问题库`

生成逻辑大致是：

1. 读取项目行业、地区、品牌名。
2. 读取已确认品牌事实。
3. 根据 GEO 分层生成问题：
   - 曝光/推荐层：用户问“哪家好”“推荐谁”“靠谱吗”。
   - 验证/口碑层：用户问“有没有资质”“是否真实”“口碑怎么样”。
   - 转化/承接层：用户问“多少钱”“怎么报名”“地址在哪”“电话多少”。
   - 权重层：用户问“行业对比”“案例”“榜单”“专业能力”。
4. 生成问题组和具体问题。

建议：

- 不要只生成宽泛问题，例如“无人机/航空哪家好”。
- 好的问题应该包含地区、行业、品牌、资质、价格、报名、地址、场景等要素。
- 如果问题生成过于潦草，先补充品牌事实库，再重新生成问题库。

## 9. 生成内容矩阵和文章

操作路径：

`左侧菜单 -> 内容管理`

常规流程：

1. 进入内容管理。
2. 在“内容任务”中选择一个任务。
3. 点击“生成草稿”。
4. 选择内容类型和发布平台。
5. 等待生成完成。
6. 到“稿件草稿”中查看文章。
7. 如果需要修改，打开草稿，在反馈区填写修改意见。
8. 系统会分析反馈并重新生成新版本。

文章生成使用的数据：

- 当前项目资料
- 已确认可公开事实
- 内容任务关联的问题
- 问题层级和用户意图
- 记忆库里的写作规则
- 当前草稿反馈
- 上一版草稿内容

注意：

- 文章生成不是简单套模板，依赖事实库和问题库。
- 如果事实库太空，文章容易泛化。
- 如果反馈写得比较口语，系统会先分析成规则，再用于改写。
- 生成文章可能需要几十秒到几分钟。

### 9.1 记录发布结果

文章发布后，可以在内容管理里记录发布结果，便于后续做信源追踪、报告归档和复测分析。

推荐流程：

1. 进入 `内容管理 -> 稿件草稿`。
2. 找到已经发布的文章。
3. 点击“记录发布”。
4. 选择一个或多个发布平台，例如官网、公众号、知乎、百家号、小红书等。
5. 如果列表里没有对应平台，可以手动输入平台名称。
6. 填写发布链接、发布时间、备注等信息。
7. 保存发布记录。

注意：

- 一篇文章可以同时记录多个发布平台。
- 如果文章已经实际发布，但系统发布检查提示存在风险，可以选择“仍然保存”。这种情况会把风险提示保留下来，方便后续复盘。
- “仍然保存”不是表示内容合规无风险，只是允许把真实发布动作记录进系统。

## 10. 记忆库怎么用

操作路径：

`左侧菜单 -> 记忆库`

记忆库用于沉淀写作偏好和长期规则，例如：

- 文章要更口语化。
- 不要出现 AI 味。
- 标题不要太夸张。
- 字数不要低于 1500 字。
- 多写本地化场景。
- 少写空泛品牌介绍，多写证据和案例。

使用方式：

1. 在草稿详情里提交反馈。
2. 系统会分析反馈，形成结构化规则。
3. 规则会进入记忆库。
4. 后续文章生成会参考这些规则。

建议：

- 反馈可以写自然语言，不需要写成提示词。
- 系统会尽量把反馈转成清晰规则。
- 如果某条规则不准确，可以在记忆库里删除或修改。

## 11. 安装 QWebBridge 浏览器插件

监测分析需要浏览器插件。它的作用是让应用可以控制你的真实浏览器：

- 打开 AI 产品网页
- 输入问题
- 点击发送
- 等待回答
- 抓取回答文本
- 回写检测结果

当前项目集成的是 QWebBridge。

### 11.1 插件目录

浏览器插件目录是：

`D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\_external\QWebBridge\packages\extension\dist`

必须选择这个 `dist` 目录。

不要选择：

- `_external\QWebBridge\packages\extension`
- `qwebbridge-extension-fixed`
- 其他临时目录

如果选择错目录，Chrome/Edge 会提示：

`清单文件丢失或不可读取`

### 11.2 Chrome 安装方法

1. 打开 Chrome。
2. 地址栏输入：

   `chrome://extensions`

3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择插件目录：

   `D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\_external\QWebBridge\packages\extension\dist`

6. 确认扩展列表里出现 QWebBridge。
7. 点击浏览器右上角扩展图标，固定该插件。

### 11.3 Edge 安装方法

1. 打开 Edge。
2. 地址栏输入：

   `edge://extensions`

3. 打开左侧或右侧的“开发人员模式”。
4. 点击“加载解压缩的扩展”。
5. 选择：

   `D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\_external\QWebBridge\packages\extension\dist`

6. 确认扩展安装成功。

### 11.4 启动 QWebBridge 服务

插件只是浏览器端，还需要本地桥接服务。

项目目录下有脚本：

`D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\启动QWebBridge-10087.cmd`

使用方法：

1. 双击 `启动QWebBridge-10087.cmd`。
2. 出现命令行窗口。
3. 窗口里应显示：

   `Starting QWebBridge on http://127.0.0.1:10087`

4. 不要关闭这个命令行窗口。
5. 再打开 GEO Flow Agent。

检查地址：

`http://127.0.0.1:10087/health`

如果正常，应看到健康检查返回信息。

### 11.5 插件显示“浏览器助手未就绪”怎么办

按下面顺序检查：

1. 是否已经双击 `启动QWebBridge-10087.cmd`。
2. 命令行窗口是否还开着。
3. 插件安装目录是否选的是 `extension\dist`。
4. 浏览器是否允许扩展在当前网页运行。
5. 当前 AI 产品网页是否已经打开并登录。
6. 刷新 AI 产品网页。
7. 刷新插件页面或重启浏览器。
8. 在 GEO Flow Agent 的监测分析页重新点击状态检查。

## 12. 监测分析怎么用

操作路径：

`左侧菜单 -> 监测分析`

推荐按最简单的流程使用：

1. 选择项目。
2. 选择目标 AI 产品，例如豆包、文心一言、Kimi 等。
3. 确认该 AI 产品网页登录状态正常。
4. 选择问题。
5. 点击开始检测。
6. 系统会通过 QWebBridge 打开网页、输入问题、等待回答、抓取答案。
7. 检测完成后查看结果。

检测结果会统计：

- 品牌是否被提及
- 是否被推荐
- 是否出现负面表述
- 是否有明确引用
- 是否与已发布信源匹配
- 样本量
- 置信等级

### 12.1 检测记录怎么看

检测完成后，可以在两个地方查看检测记录：

1. `左侧菜单 -> 监测分析 -> 检测记录`
2. `项目管理 -> 打开具体项目 -> 检测记录`

项目详情里的检测记录已经改成更直观的展示方式：

- `检测方式`：显示本次是 `单题检测`、`批量检测`，还是 `未采样`。
- `检测平台`：显示本次检测对应的 AI 产品，例如豆包、文心一言、Kimi 等。
- `样本数`：显示本次检测实际抓取了几个回答样本。
- `机制`：显示本次检测使用的 GEO 机制类型，例如 B。
- `采样策略`：显示 MVP 抽样、全量检测或自定义策略。
- `状态`：显示运行中、已完成、失败等状态。

判断方法：

- 样本数为 `1`：通常是单题检测。
- 样本数大于 `1`：通常是批量检测。
- 检测平台显示具体名称：说明这条记录可以追溯到对应 AI 产品。
- 检测平台显示“未绑定平台”：说明旧记录或手工记录没有保存平台 ID。

建议：

- 每个平台可以单独跑一轮。
- 例如先跑豆包，再跑文心，再跑 Kimi。
- 后续在报告中心可以多选这些检测记录，聚合生成一份报告。

## 13. 配置 AI 产品网页选择器

有些 AI 网页结构会变，导致自动点击发送失败。

如果出现类似提示：

`问题已输入到网页，但输入框仍保留该问题，说明没有成功发送。请在检测平台为该 AI 产品配置 submit_selector`

说明系统找到了输入框，但没有点中发送按钮。

处理方法：

1. 到监测平台/目标 AI 产品配置处。
2. 找到该产品的 `submit_selector` 字段。
3. 把系统提示里最像发送按钮的 selector 填进去。
4. 保存。
5. 重新检测。

文心一言常见发送按钮可能类似：

`[class*="send__"]`

如果页面改版，需要重新识别。

## 14. 报告中心怎么用

操作路径：

`左侧菜单 -> 报告中心`

使用方式：

1. 选择项目。
2. 选择报告类型：
   - 客户报告
   - 内部报告
3. 选择检测记录。
4. 如果只选一组检测记录，可以选择基线做对比。
5. 如果选择多组检测记录，系统会生成聚合报告。
6. 点击生成报告。
7. 检查报告内容。
8. 点击保存归档。

多组检测聚合适用场景：

- 你一次只检测了一个平台。
- 例如：
  - 豆包一组
  - 文心一言一组
  - Kimi 一组
- 想合并成一份整体 GEO 检测报告。

注意：

- 多组聚合时，基线对比会自动禁用。
- 因为多组聚合和单组基线对比统计口径不同。
- 如果要做验收对比，建议使用单组检测记录对比单组基线。

## 15. 常见问题

### 15.1 生成文章失败：timeout of 30000ms exceeded

旧版本前端默认 30 秒超时，长文生成容易触发。

当前版本已修复：

- 文章生成接口超时提高到 300 秒。
- 后端内容生产 Agent 调用模型也提高到 300 秒。

如果仍然超时：

1. 检查模型 API 是否可用。
2. 换响应更快的模型。
3. 降低文章字数。
4. 检查网络。
5. 避免一次让模型生成过长、过复杂的文章。

### 15.2 文章内容很空、像模板

通常是事实库不足。

处理方法：

1. 回到品牌事实库。
2. 粘贴更多企业资料。
3. 用 AI 提取事实。
4. 人工确认可公开事实。
5. 再重新生成文章。

### 15.3 文章里出现 `[品牌名]`、`[行业]` 这种占位符

说明模型没有拿到足够明确的事实，或输出没有遵守格式。

处理方法：

1. 补齐品牌名、公司主体、地区、行业。
2. 确认可公开事实。
3. 在反馈里写：“不要出现任何方括号占位符，必须用真实企业信息替换。”
4. 重新生成。

### 15.4 WebBridge 返回 502

常见原因：

1. QWebBridge 服务没启动。
2. 浏览器插件没安装或没连接。
3. 插件目录选错。
4. AI 网页没登录。
5. 目标网页还在加载、验证或风控。
6. 页面结构变化，选择器失效。

处理顺序：

1. 双击 `启动QWebBridge-10087.cmd`。
2. 打开 `http://127.0.0.1:10087/health`。
3. 检查浏览器插件是否显示已连接。
4. 打开目标 AI 网页并登录。
5. 刷新网页。
6. 重新运行检测。

### 15.5 插件安装提示“清单文件丢失”

目录选错了。

正确目录：

`D:\新版GEO优化项目\aio-marketing-suite\geo-flow-agent-v2\geo-flow-agent-v2\_external\QWebBridge\packages\extension\dist`

这个目录里必须能看到：

- `manifest.json`
- `background.js`
- `popup.html`
- `popup.js`

### 15.6 文心一言输入了问题但不发送

可能是发送按钮选择器失效。

处理方法：

1. 在检测平台中为文心一言配置 `submit_selector`。
2. 可优先尝试：

   `[class*="send__"]`

3. 确保网页输入框里有内容，发送按钮不是灰色。
4. 重新检测。

### 15.7 检测时一个问题失败后还继续打开下一题

当前逻辑已经尽量避免这个问题。如果仍出现：

1. 检查是否使用最新打包版本。
2. 检查 QWebBridge 插件是否为当前项目目录里的 `dist` 版本。
3. 检查目标 AI 产品配置。

### 15.8 杀毒软件误删应用

原因通常是：

- 本地打包程序没有代码签名。
- PyInstaller 后端 exe 容易被部分杀毒软件误判。

处理方法：

1. 把 `release` 目录加入杀毒软件信任区。
2. 使用安装版重新安装。
3. 不要从临时目录运行。
4. 如果仍被删，需要在杀毒软件里恢复并添加白名单。
