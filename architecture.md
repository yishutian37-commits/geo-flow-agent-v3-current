# GEO Flow Agent V2.3 架构说明

## 1. 系统定位

GEO Flow Agent V2.3 是一个面向 GEO/AIO 场景的本地化 AI 工作流应用。它把企业资料、项目知识库、品牌事实库、问题矩阵、多平台 AI 监测、内容任务、发布检查、报告和经验技能沉淀串成一个可复用闭环。

当前系统更准确的定位是：

> 本地化 AI 工作流产品 + 多个半自动 Agent 能力模块。

它已经具备完整业务链路，主控 Orchestrator 已能对诊断、生产、监测三条主链路做轻量预检和状态反馈；后续仍可继续增强为更强的自主调度层。

## 2. 技术栈

- 前端：React、Ant Design、Axios。
- 后端：FastAPI、SQLAlchemy Async、SQLite/Postgres、Pydantic。
- 桌面端：Electron、PyInstaller、electron-builder。
- AI 模型：OpenAI 兼容接口配置，支持文本模型与可选视觉模型。
- 浏览器桥接：QWebBridge 与 Kimi WebBridge，用于真实 AI 网页自动提问、抓取回答、截图和来源。
- 测试与验证：pytest、React build、GitHub Actions。

## 3. 核心闭环

```text
项目创建
  -> 项目知识库/语料库
  -> 品牌事实库
  -> 问题矩阵
  -> 多平台 AI 检测
  -> 回答/截图/来源留档
  -> 内容任务
  -> 草稿生成
  -> 发布检查
  -> 发布记录
  -> 复测报告
  -> 经验技能库回流
```

## 4. 主要数据对象

- `Project`：项目基础信息、行业、地区、目标 AI 产品。
- `CorpusItem`：项目资料和语料库条目，用于 AI 分层入库和事实抽取。
- `Brand` / `BrandFact`：品牌主体和已确认事实，约束内容生成。
- `QuestionGroup` / `Question`：问题矩阵，承载意图层、关键词拆解、启停、优先级和平台建议。
- `ModelTarget`：检测平台配置，包括网页地址、选择器、识别模式、视觉能力。
- `MonitoringRun` / `MonitoringSample`：检测批次和问题 × 平台明细。
- `ContentTask` / `ContentDraft`：内容任务和草稿版本。
- `PublishRecord`：一文多平台发布记录。
- `ExperienceSkill`：从人工反馈中沉淀的项目级、行业级、全局级经验技能。

## 5. Agent 与服务分工

- `ProductionAgent`：负责事实约束的内容生成、合规检查、平台检查、发布前检查。
- `StrategyAgent`：负责从诊断结果中判断内容触发点和策略建议。
- `ProjectAgent`：负责项目体检相关逻辑。
- `Orchestrator`：负责工作流入口、状态流转、发布检查，以及诊断/生产/监测三条主链路的前置条件检查、阻断原因和下一步建议。
- `project_knowledge_service`：负责项目资料入库、AI 分析和知识资产沉淀。
- `webbridge_service`：负责真实网页检测、输入、发送、回答抓取、视觉兜底、截图和来源提取。

## 6. 事实约束机制

内容生成优先读取已确认、公开、未过期的品牌事实。对于资质、价格、地址、案例、联系方式、通过率等高风险事实，系统要求来自事实库，不鼓励模型自由发挥。

当前事实校验主要包括：

- 已确认公开事实注入；
- 高风险表达检查；
- 发布检查；
- 平台政策检查；
- 基于事实文本的引用识别。

后续增强方向：

- claim 抽取；
- claim 与事实库逐条对齐；
- 同义改写匹配；
- contradiction 判断；
- 高风险声明人工确认。

## 7. 监测链路

监测分析以“问题 × 平台”为基本粒度。系统会在真实 AI 网页中提问，并保存：

- 问题；
- 平台；
- 原始回答；
- 是否提及品牌；
- 是否推荐品牌；
- 情绪判断；
- 信息来源；
- 回答截图；
- 检测时间；
- 失败原因。

默认使用文本抓取模式。只有检测平台明确开启视觉识别时，才会调用视觉模型分析截图。

## 8. 经验技能回流

写作反馈和修改意见不会直接当作规则使用，而是先由 AI 分析成待确认技能建议。确认后可以沉淀为：

- 项目级技能；
- 行业级技能；
- 全局级技能。

文章生成时会按项目、行业、全局顺序读取技能，但技能不能突破已确认事实边界。

## 9. 启动与账号初始化

新环境默认走首次启动初始化管理员账号：

1. 数据库中没有任何用户时，登录页显示初始化管理员表单。
2. 用户自行设置管理员用户名、邮箱和密码。
3. 系统不再默认创建固定密码的项目负责人账号。

开发环境如需自动创建项目负责人账号，必须显式设置：

```text
GEO_SEED_DEFAULT_PROJECT_OWNER=1
GEO_DEFAULT_PROJECT_OWNER_USERNAME=pm01
GEO_DEFAULT_PROJECT_OWNER_EMAIL=pm01@geoflow.app
GEO_DEFAULT_PROJECT_OWNER_PASSWORD=<your-local-password>
```

## 10. 当前已知边界

- Orchestrator 已具备轻量预检编排能力，但还没有自动串联完整 Agent 执行链、异步任务队列和失败恢复重试。
- 平台适配已具备规则检查和提示，但平台化重写还需要继续增强。
- 浏览器自动化依赖目标网页结构和登录态，必须保留人工补录与失败留证兜底。
- 事实校验已有边界意识，但还不是严格的自然语言 claim 校验系统。
