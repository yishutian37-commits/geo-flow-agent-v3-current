# GEO Flow Agent V2.3 Agent 介绍

## 一句话介绍

GEO Flow Agent V2.3 是一套面向生成式搜索和 AI 问答场景的本地化 GEO/AIO 优化工作台，帮助企业把项目知识资产、品牌事实、用户问题、内容生产、AI 平台监测、信息来源分析、发布复测和报告沉淀串成一个闭环。

## 为什么需要它

越来越多用户不再只通过传统搜索引擎找答案，而是直接向豆包、文心一言、Kimi、通义、DeepSeek、ChatGPT 等 AI 产品提问：

- 包头无人机执照培训哪家靠谱？
- 某公司有没有正规资质和证书编号？
- 某品牌的课程价格真实吗？
- 本地哪家机构更适合报名？
- 某企业和同行相比有什么优势？

这意味着企业的线上可见度不只取决于网页排名，也取决于 AI 在回答问题时是否：

1. 知道品牌。
2. 愿意提及品牌。
3. 能准确引用企业事实。
4. 会把品牌放入推荐列表。
5. 不产生错误、空泛或负面回答。

GEO Flow Agent 的目标，就是围绕这些 AI 回答场景，系统化建设「可被 AI 识别、采信和推荐」的内容资产。

## 它解决的核心问题

### 1. 企业资料散乱，AI 不知道该信什么

很多企业资料分散在官网、公众号、PPT、PDF、资质证书、销售话术和新闻报道中。直接让模型写文章，很容易出现空泛内容、占位符、编造资质或遗漏关键地区信息。

GEO Flow Agent 通过品牌事实库把资料拆成可审核事实，例如：

- 企业主体
- 品牌名称
- 所属行业
- 服务地区
- 资质证书
- 证书编号
- 地址
- 联系方式
- 产品/课程
- 价格
- 案例
- 荣誉
- 官网和媒体报道

只有确认后的公开事实，才会进入文章生成、发布检查和报告引用。

### 2. 不知道真实用户会怎么问 AI

传统 SEO 关注关键词，GEO 更关注自然语言问题。系统会基于行业、地区、品牌、产品、资质、价格、案例等信息生成问题库。

问题库按层级组织：

- 曝光/推荐层：哪家好、推荐谁、靠谱吗。
- 验证/口碑层：有没有资质、证书真假、评价如何。
- 转化/承接层：多少钱、怎么报名、地址在哪、电话多少。
- 权威/对比层：行业对比、案例、榜单、专业能力。

每条问题还会带上关键词拆解、问题公式、商业价值、证据支撑、内容可执行性和推荐发布平台，方便后续内容任务和监测分析。

### 3. 内容生产和问题库脱节

普通写作工具通常只解决“写一篇文章”。GEO Flow Agent 的内容生产逻辑是：

```text
问题库 -> 内容任务 -> 事实引用 -> 平台规则 -> 文章草稿 -> 发布检查 -> 发布记录 -> 复测
```

每篇文章都应该服务于某类 AI 问答场景，而不是孤立生成。

### 4. 写作反馈无法沉淀

用户常见反馈包括：

- 不要太像 AI 写的。
- 多写本地化场景。
- 字数不要低于 1500 字。
- 标题不要夸张。
- 多引用资质和证据。
- 少写空话，多写案例。

系统会把自然语言反馈交给 AI 分析，转成可复用写作规则，进入记忆库；其中更适合长期复用的方法论，会形成“经验技能建议”，由用户确认后进入经验技能库。后续文章生成和改写会参考这些规则，而不是机械照抄用户原话。

### 5. 不知道 AI 平台实际怎么回答

GEO 优化不能只看“发了多少文章”，还要检测 AI 产品到底如何回答用户问题。

系统通过 WebBridge / QWebBridge 控制真实浏览器页面，自动完成：

1. 打开目标 AI 产品网页。
2. 输入问题。
3. 点击发送。
4. 等待回答生成。
5. 抓取原始回答文本。
6. 判断是否提及品牌。
7. 判断是否推荐品牌。
8. 判断情绪和风险。
9. 抓取回答中的信息来源链接。
10. 保存回答截图。

检测结果会转为可读的明细和统计指标，而不是只返回 JSON。

## 核心模块

### 项目管理

管理不同客户或品牌项目。项目是数据边界，品牌事实、问题库、内容任务、监测记录和报告都归属到具体项目。

### 品牌事实库

用于沉淀可信资料源。支持一次性粘贴企业资料，由 AI 抽取候选事实，再人工确认。事实会区分草稿、已确认、公开可用、限制使用、过期和争议状态。

### 项目知识库

项目知识库保存比品牌事实更宽的资料资产，包括基础信息、案例故事、判断逻辑、竞品/差评、内容素材和复盘数据。它不替代品牌事实库，尤其是资质、价格、地址、证书编号等确定性信息仍需以已确认公开事实为准。

项目知识库的价值在于让资料可以被反复调用：问题矩阵可以参考判断逻辑，内容任务可以引用案例和素材，监测结果可以沉淀成复盘资料，下一轮内容优化又能继续使用这些经验。

### 资料缺口诊断

检查项目是否缺少企业主体、品牌名、资质编号、地址、价格、产品、案例、荣誉、联系方式等关键内容，帮助运营人员知道下一步该补什么资料。

### 问题库

模拟真实用户会向 AI 提出的问题。问题不是普通关键词，而是围绕“推荐、验证、转化、权威”的问题矩阵。

系统内置行业问题模板库，用来控制不同行业的问题主体称呼、典型问法、行业禁用词、正向样例和反向样例。这样问题矩阵不再只适配某一个历史项目，而是可以逐步扩展到制造、消费、本地生活、技术服务、专业服务等更多行业。

当运营人员人工新增、改写、禁用或删除问题时，系统会记录这些调整，并在系统设置里生成“问题模板优化建议”。管理员确认后，建议才会写入行业模板库，影响后续新生成的问题矩阵。

### 内容管理

包含内容任务、稿件草稿、发布检查、发布渠道和发布记录。系统会基于问题库、事实库、项目知识资产、平台规则和记忆库生成平台化草稿。新建内容任务时可以手动选择项目知识资产；如果不选，系统会根据关联问题的知识需求自动推荐相关资料。

### 平台规则库

内置不同自媒体平台的标题、字数、引流、AIGC 标识和风险表达规则。规则支持在系统设置中编辑，后续生成文章和发布检查都会参考。

### 提示词模板库

GEO 关键 prompt 已从代码中抽离到模板文件，包括问题生成、推荐逻辑、内容规划、文章生成和反馈改写记忆，方便后续维护升级。

### 记忆库

保存项目写作偏好和长期规则。用户反馈不直接当规则，而是由 AI 分析成更清晰的写作约束，再参与后续改写。

### 经验技能库

保存“怎么做”的可复用方法，不保存企业事实本身。它适合沉淀文章反馈、问题库编辑、监测复盘和平台发布检查中反复出现的方法经验。

当前版本支持文章反馈、监测复盘和发布检查自动生成待确认技能建议。用户确认后，技能可以作为项目级、行业级或全局规则启用；后续文章生成会自动读取命中的启用技能，用来影响表达风格、结构组织、证据呈现和平台适配。技能不是一次启用后就写死，后续可以编辑、停用、删除、升级或降级作用域，并通过版本历史回滚旧版本。但资质、价格、地址、证书编号、案例结果等确定性事实仍必须来自已确认品牌事实库。

### 监测分析

按「问题 × 平台」记录 AI 实际回答，展示提及率、推荐率、负面提及、来源覆盖、原始回答、截图和信息来源。

监测明细可以直接删除，也可以把未提及/未推荐的问题一键转成内容任务。AI 搜索详情支持把单条检测样本沉淀为复盘资料，保存到项目知识库，用于后续内容优化和问题矩阵迭代。

### 报告中心

支持单次检测报告，也支持选择多组检测记录生成聚合报告，适合对比多个平台、多个批次或阶段性优化效果。

## 和普通写作工具的区别

普通写作工具解决的是：

```text
输入主题 -> 生成文章
```

GEO Flow Agent 解决的是：

```text
企业资料 -> 品牌事实 -> 问题矩阵 -> AI 监测 -> 内容任务 -> 平台稿件 -> 发布记录 -> 复测报告
          ↘ 项目知识库（资料、素材、判断逻辑、复盘数据） ↗
          ↘ 经验技能库（写作规则、问题公式、复盘方法） ↗
```

它更像一个 GEO 项目操作系统，而不是单一文案工具。

## 当前版本亮点

- 支持品牌事实批量提取。
- 支持项目知识库分层管理，沉淀基础信息、案例故事、判断逻辑、内容素材和复盘数据。
- 支持经验技能库，把人工反馈、监测复盘和发布检查沉淀成待确认技能，确认后参与后续文章生成。
- 支持经验技能版本历史、修订原因、作用域调整和旧版本回滚。
- 支持平台规则库和规则编辑。
- 支持行业问题模板库，问题矩阵可按行业配置。
- 支持问题模板学习闭环，人工调整问题后可沉淀为模板优化建议。
- 支持平台化文章生成。
- 支持一篇文章多平台发布记录。
- 支持发布风险提示后继续保存真实记录。
- 支持文本抓取和可选视觉识别检测。
- 支持 AI 搜索详情、原始回答、截图和信息来源。
- 支持检测明细一键生成内容任务。
- 支持 AI 搜索详情一键沉淀为复盘资料，回流项目知识库。
- 支持项目详情展示 GEO 知识闭环统计。
- 支持多组检测记录聚合报告。
- 支持项目负责人账号和角色权限。

## 数据与安全特点

- 本地桌面应用运行，默认使用本地 SQLite 数据库。
- 项目数据相互隔离，避免客户资料混用。
- 事实需要人工确认后才进入正式生成和引用。
- 删除检测记录、检测明细等破坏性操作需要角色权限。
- 浏览器桥接使用用户自己的浏览器登录态，不直接保存第三方 AI 平台账号密码。

## 当前定位

GEO Flow Agent V2.3 已经具备从项目资料到 AI 监测、内容优化和报告复盘的完整主链路。后续更适合继续增强：

- 发布后自动安排复测。
- 来源资产权重评分。
- 竞品来源监测。
- 平台风险趋势统计。
- 自动从检测结果生成内容计划。

当前版本已经开始具备“越用越好用”的基础：问题库的人工修改不会只停留在单个项目里，而是可以经管理员确认后回流到行业模板库；文章反馈也不会只停留在一次改稿里，而是可以经用户确认后沉淀成经验技能，成为后续同类写作和内容优化的可复用经验。
